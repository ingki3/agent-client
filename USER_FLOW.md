# Agent Client — User Flow & Screen Design (MVP)

> 이 문서는 [PRD.md](./PRD.md)의 MVP 범위(전화번호 로그인 · 친구 추가 · 친구 리스트 · 스트리밍 채팅 · thinking/tool-call trace)에 한정해 작성된 **화면 단위 설계도**다. 기반 규칙은 전부 Telegram(Bot API + 표준 인증 흐름) 기준이며, 그 위에 GFM 마크다운 / 스트리밍 / trace 가시화 확장만 얹는다.

## 0. 문서 사용 규약

### 0.1 ID 네이밍
- `S-XX` : 일반 화면(Normal Screen, 풀 페이지)
- `M-XX` : 모달/바텀시트(Modal / Bottom Sheet)
- `D-XX` : 다이얼로그/팝업/토스트(Dialog / Popup / Snackbar)
- `I-XX` : 인라인 컴포넌트(채팅 메시지에 부착되는 요소)

### 0.2 상태 표기
- `[GUEST]` : 로그인 안 됨(앱 첫 실행 ~ 인증 완료 전)
- `[AUTH]`  : 로그인 완료 — 일반 사용 가능
- `[OFFLINE]` : `[AUTH]` 중 네트워크 단절 (Read-only 캐시 모드)
- `[ANY]`  : 상태와 무관 (Splash 등)

> **규칙**: 동일 화면이라도 상태가 달라지면 별도 ID 또는 명시적 분기(`-EMPTY` 등)로 분리한다.

### 0.3 화면 표 형식

| 필드 | 설명 |
|------|------|
| ID | 위 네이밍 규약 |
| Name | 화면 표시명 |
| Type | Normal / Modal / BottomSheet / Dialog / Snackbar / Inline |
| State | 진입 가능한 사용자 상태 |
| Info | 화면에서 노출되는 정보(데이터 항목) |
| Actions | 사용자 입력(버튼/링크/제스처) → 이동 대상 ID 또는 동작 |
| Notes | 엣지케이스, 빈 상태, 에러 처리 |

## 1. 상태 머신 (User State Machine)

```
[설치] ─► [GUEST] ──(전화번호 + SMS 코드 인증 성공)──► [AUTH]
                                                          │
                                                          ├──(네트워크 단절)──► [OFFLINE]
                                                          │                       │
                                                          │                       └──(재연결)──► [AUTH]
                                                          │
                                                          └──(로그아웃)──► [GUEST]
```

전이 규칙
- `[GUEST] → [AUTH]`: S-03에서 SMS 코드 검증 성공
- `[AUTH] → [OFFLINE]`: 네트워크 감시자가 단절 감지 → D-06 배너 노출 + 송신 큐 보류
- `[AUTH] → [GUEST]`: S-20 설정에서 로그아웃 확정 시(D-05) — 인증 토큰·봇 토큰·로컬 캐시 삭제

## 2. 화면 카탈로그

### 2.1 Onboarding & Auth — `[GUEST]`

#### S-01 · Splash
| 필드 | 내용 |
|------|------|
| Type | Normal |
| State | `[ANY]` |
| Info | 로고, 버전, 로딩 인디케이터 |
| Actions | (자동) 초기화 완료 → 상태 분기: `[GUEST]`이면 S-02, `[AUTH]`이면 S-10 |
| Notes | 초기화 실패(네트워크 등) 시 재시도 토스트 |

#### S-02 · 전화번호 입력
| 필드 | 내용 |
|------|------|
| Type | Normal |
| State | `[GUEST]` |
| Info | 국가 코드 선택(드롭다운, 기본: 단말기 지역 추정), 전화번호 입력 필드, 약관·개인정보 동의 체크박스(1줄 인라인) |
| Actions | [다음] → `POST /auth/send-code` → 성공 시 S-03, 실패 시 인라인 에러 · 국가 코드 탭 → 국가 검색/선택 인라인 시트 · 약관 링크 탭 → 외부 브라우저 |
| Notes | 동의 미체크 시 [다음] 비활성. 잘못된 번호 형식이면 즉시 인라인 에러 |

#### S-03 · SMS 코드 입력
| 필드 | 내용 |
|------|------|
| Type | Normal |
| State | `[GUEST]` |
| Info | 발송 번호 표시, 5~6자리 코드 입력(OTP 자동 채움 지원), 남은 시간 카운트다운, [재전송] / [음성 통화로 받기] 보조 액션 |
| Actions | 코드 입력 완료 → `POST /auth/verify-code` → 성공 시 S-10, 실패 시 D-01 · [재전송] → `send-code` 재호출 후 타이머 리셋 · [음성 통화] → 동일 엔드포인트, 채널=voice · [번호 수정] → S-02 복귀 |
| Notes | 만료(예: 5분) 시 입력 비활성 + [재전송] 강조 |

### 2.2 메인 — `[AUTH]`

#### S-10 · 친구 리스트 (Home)
| 필드 | 내용 |
|------|------|
| Type | Normal |
| State | `[AUTH]` (등록 친구 0개일 때 S-10-EMPTY 변형) |
| Info | 상단바(앱 이름·[설정]), 친구 카드 리스트(아이콘·표시명·마지막 메시지 미리보기·시각·미확인 카운트·연결 상태), FAB [+] |
| Actions | 카드 탭 → S-11(해당 친구 채팅) · 카드 길게 누름 → M-02(친구 빠른 액션) · FAB [+] → S-12(친구 추가) · 우상단 [설정] → S-20 · 당겨서 새로고침 → `getUpdates` 수동 트리거 |
| Notes | `[OFFLINE]`이면 상단 D-06 배너 고정. 마지막 메시지·미확인 카운트는 로컬 캐시 기반 |

#### S-10-EMPTY · 친구 리스트 (빈 상태)
| 필드 | 내용 |
|------|------|
| Type | Normal |
| State | `[AUTH]`, 등록 친구 0 |
| Info | "아직 등록된 친구가 없어요" 일러스트, 짧은 안내(봇 토큰으로 추가) |
| Actions | [+ 친구 추가] → S-12 · 우상단 [설정] → S-20 |

#### S-11 · 채팅 화면 (Conversation)
| 필드 | 내용 |
|------|------|
| Type | Normal |
| State | `[AUTH]` 또는 `[OFFLINE]` (송신은 큐잉) |
| Info | 상단바(친구 아이콘·이름·연결 상태·[←]) / 메시지 타임라인(말풍선·타임스탬프·날짜 구분선·상태 아이콘) / I-01 Trace 패널(에이전트 응답에 부착) / 입력바(텍스트 필드·전송) / 응답 스트리밍 중 [Stop] 버튼·스크롤-다운 버튼 |
| Actions | 메시지 입력 → [전송] (상태: 전송 중 → 전송됨 → 응답 중(스트리밍) → 완료) · 송신 실패 시 메시지에 ⚠ + 길게 눌러 D-02 · 응답 중 [Stop] 탭 → 스트리밍 중단 요청 · 상단 [←] → S-10 · 스크롤로 과거 메시지 조회(로컬 캐시 한도 내) |
| Notes | 응답이 도착하기 시작하면 봇 측 버블이 즉시 생기고 `message_delta` 청크가 누적됨. 마크다운은 스트리밍 중에도 안전 점진 렌더링(미완성 토큰은 노출 안 함). `[OFFLINE]`이면 입력바 placeholder가 "오프라인 — 연결되면 전송됩니다", 전송 시 큐에 저장 |

#### I-01 · Trace 패널
| 필드 | 내용 |
|------|------|
| Type | Inline (S-11 내, 에이전트 응답 메시지 하단 부착) |
| State | `[AUTH]` |
| Info | **접힘**: "🧠 N단계 사고 · 🛠 M개 툴 호출 · ⏱ 총 t초" 한 줄 요약. **펼침**: thinking 노드(plan/요약 텍스트), tool_call 노드(이름·인자 요약·결과 요약·latency) 시퀀스 |
| Actions | 헤더 탭 → 펼침/접힘 토글 · 노드 탭 → M-01(원본 JSON) |
| Notes | 응답 스트리밍 중에는 노드가 실시간으로 추가되고 스피너 표시. 트레이스 이벤트가 비어 있으면 패널 미노출(표준 텔레그램 봇 fallback). 인자 JSON에 민감값 마스킹 |

### 2.3 친구 관리 — `[AUTH]`

#### S-12 · 친구 추가 — 봇 토큰 입력
| 필드 | 내용 |
|------|------|
| Type | Normal (또는 BottomSheet 형태 가능) |
| State | `[AUTH]` |
| Info | 봇 토큰 입력 필드(마스킹 토글), [붙여넣기] 버튼, "토큰은 어디서 얻나요?" 도움말 링크 |
| Actions | [다음] → `getMe` 호출 → 성공 시 S-13, 실패 시 인라인 에러("유효하지 않은 토큰입니다") · [붙여넣기] → 클립보드 삽입 · [←] / [닫기] → S-10 |
| Notes | 동일 토큰이 이미 등록되어 있으면 즉시 D-03 |

#### S-13 · 친구 추가 — 미리보기 / 확정
| 필드 | 내용 |
|------|------|
| Type | Normal (또는 S-12와 같은 stack의 다음 단계) |
| State | `[AUTH]` |
| Info | `getMe` 결과 미리보기(봇 아이콘·이름·`@username`·설명), 표시명 편집 필드(기본값: 봇 이름) |
| Actions | [추가] → 친구 레코드 생성 + 봇 토큰 SecureStore 저장 + `/start` 자동 전송 → S-11(새 채팅) · [←] → S-12 |
| Notes | 등록 직후 `/start` 응답이 오기까지 입력바는 활성 |

## 3. 주요 사용자 여정 (Flow Diagrams)

### 3.1 신규 가입 (First Launch)
```
S-01 → S-02 (전화번호) → S-03 (SMS 코드) → S-10-EMPTY
                                          → [+ 친구 추가]
                                          → S-12 → S-13 → S-11 (자동 /start)
```

### 3.2 자동 로그인 (Returning User)
```
S-01 → (인증 토큰 유효) → S-10
```

### 3.3 친구 추가 (UC-02)
```
S-10 FAB[+] → S-12 (토큰 입력)
            → getMe 성공 → S-13 (미리보기) → [추가] → S-11
            → getMe 실패 → S-12 인라인 에러
            → 중복 감지 → D-03 → [기존 친구 열기 → S-11] 또는 [취소 → S-12]
```

### 3.4 채팅 (UC-03, 스트리밍 + 마크다운)
```
S-10 → 친구 카드 탭 → S-11 → 메시지 입력 → [전송]
   ▶ 사용자 버블 (상태: 전송 중 → 전송됨)
   ▶ 봇 버블 생성 (상태: 응답 중)
       → message_delta 청크 누적 (점진 렌더, 마크다운 안전 점진 적용)
       → I-01 trace 노드 실시간 추가
   ▶ 봇 버블 완료 (상태: 완료, 최종 마크다운 서식 적용)
실패 시 사용자 버블에 ⚠ → 길게 누름 → D-02 (재전송)
스트리밍 중 [Stop] 탭 → 중단 요청
```

### 3.5 Trace 확인 (UC-04)
```
S-11 응답 도착 → I-01 접힘 요약 → 사용자 탭 → I-01 펼침
   → thinking / tool_call / tool_result 노드 시퀀스 렌더
   → 노드 탭 → M-01 (원본 JSON) → [닫기] → S-11
```

### 3.6 친구 삭제
```
S-10 → 카드 길게 누름 → M-02 → [삭제] → D-04 → 확정 → S-10 갱신
```

### 3.7 로그아웃 (UC-01)
```
S-10 → S-20 → [로그아웃] → D-05 → 확정
   → 인증·봇 토큰·로컬 캐시 삭제 → 상태 [AUTH]→[GUEST] → S-01 → S-02
```

### 3.8 오프라인 ↔ 온라인 사이클
```
S-11 사용 중 → 네트워크 단절 → D-06 배너 + 입력 큐잉
            → 재연결 → 큐 자동 송신, 상태 아이콘 동기화
```

## 4. 화면 연결성 검증 (Connectivity Audit)

목표: **모든 화면이 진입 경로와 출구 경로를 가진다.**

### 4.1 진입 경로 요약

| 화면 | 진입 경로 |
|------|-----------|
| S-01 | 앱 실행 |
| S-02 | S-01(GUEST), S-03 [번호 수정] |
| S-03 | S-02 [다음] 성공 |
| S-10 / S-10-EMPTY | S-01(AUTH), S-03 검증 성공, S-11 [←], S-13 [추가] 완료, D-04/D-05 확정 후 |
| S-11 | S-10 카드, S-13 추가 완료, D-03 [기존 친구 열기] |
| S-12 | S-10 FAB[+], S-10-EMPTY CTA |
| S-13 | S-12 `getMe` 성공 |
| S-20 | S-10 [설정] |
| S-21 | S-20 [정보] |
| I-01 | S-11 메시지 단위 |
| M-01 | I-01 노드 |
| M-02 | S-10 카드 길게 누름 |
| D-01 | S-03 verify 실패 |
| D-02 | S-11 실패 메시지 길게 누름 |
| D-03 | S-12 `getMe` 결과가 기존 친구 |
| D-04 | M-02 [삭제] |
| D-05 | S-20 [로그아웃] |
| D-06 | 네트워크 단절(자동) |

### 4.2 출구(Back) 규칙
- 모든 `S-` 화면은 시스템 뒤로가기 / 좌상단 [←]로 직전 화면(또는 S-10)으로 복귀.
- 모든 `M-` / `D-`는 [닫기] / [취소] 또는 외부 영역 탭으로 호출 화면 복귀.

### 4.3 데드엔드 점검
- ✅ 모든 진입 화면에서 S-10(홈) 또는 S-01(로그아웃 후)로 도달 가능.
- ✅ 인증 실패(D-01)는 항상 S-03 재시도 또는 S-02 복귀 경로 보유.
- ✅ `getMe` 실패는 S-12 인라인 에러 → 재입력 가능.
- ✅ `[OFFLINE]`에서도 S-11 입력은 큐잉, 재연결 시 자동 송신.
- ✅ 친구 삭제 / 로그아웃 후에도 일관된 빈 상태(S-10-EMPTY / S-01) 복귀.

## 6. 컴포넌트 디자인 토큰 (참고)

> Pencil 등으로 와이어프레임을 만들 때 일관성을 위한 최소 토큰. 시각 디자인은 별도 결정.

- **컬러 슬롯**: `primary`, `surface`, `surface-elevated`, `border`, `text-primary`, `text-secondary`, `bubble-user`, `bubble-agent`, `bubble-system`
- **버블 종류**: `user` / `agent` / `system`(예: `/start` 자동, 친구 추가 알림)
- **메시지 상태 아이콘**: `sending` · `sent` · `streaming` · `done` · `failed` · `queued-offline`
- **Trace 노드 아이콘**: `thinking`(🧠) · `tool_call`(🛠) · `tool_result`(✅/⚠)
- **마크다운 렌더 슬롯**: heading 1~6, bold/italic/strike, inline code, code block(언어 하이라이트), ordered/unordered list, nested list, checkbox, table, blockquote, link, image, hr
